About our Project

Welcome to our Project!

Today, melanoma remains the third most common type of skin cancer worldwide. As such, healthcare professionals seek to find a way to treat it as early as possible before it becomes fatal. In this project, we aim to utilize coding languages such as Python and HTML to predict the size of a melanoma tumor based upon its characteristics. It is our hope that our model can help healthcare professionals identify melanoma tumors as early as possible so that it can be treated with no further complications.

Goals

What we aim to do with our project

Discover what determines melanoma tumor size

Use graphics to determine how nine different factors affect the size of the tumor

Train a machine learning model

Predicting the size of melanoma tumors using machine learning will be useful as there are different risks associated with different tumor sizes.

Aid the Healthcare Industry

Create a simple, easy-to-use website that can detect patterns within different factors that determine the size of a melanoma tumor

MVP

Minimum Viable Product

In 3 weeks, our product will have graphs and charts of characteristics such as the damage size, exposed area, and tumor size of melanoma tumors, and our product will be able to make accurate predictions of the size of future melanoma tumors based on these characteristics.

Tech Stack

Tech we will need in order to create our project

This project will use HTML, CSS, JavaScript and other programming languages to provide a clean, easy to use, intuitive GUI to interface with a Python backbone. We also used Pandas for data management, Numpy for mathematical operations, SKLearn to make the machine learning model, and Plotly and Seaborn for graphing our data.

Machine Learning

Our machine learning models

Regression vs. Classification

Since this problem is trying to predict a value rather than classify an input, it is classified as a regression problem. Since the value we are trying to predict, tumor size, is continuous, this project will be solving a regression problem.


Supervised vs. Unsupervised

Because we fed our model data that was already labeled, this problem is an example of supervised machine learning.


Our Top 3 Machine Learning Models

We tested out five models (K Nearest Neighbors, Support Vector Regression, Decision Tree Regressor, Linear Ridge Regression, and Gradient Boost Regressor) and found these 3 to be the best for our data.

Linear Ridge Regression

Ridge regression is a method of estimating the coefficients of multiple-regression models. Ridge regression is used when you have data in which multiple independent variables are correlated

Gradient Boosting Regressor

Gradient Booster Regressor finds the difference between the current prediction and known target value.Trains a weak model to correlate to that target value.

Decision Tree Regressor

The Decision Tree Regressor shapes the data in the form of a tree-like structure. Breaks down the dataset into smaller subsets and mainly aids in solving classification problems.

Comparisons

Deciding on a machine learning model


Root Mean Squared Error

RMSE measures the differences between the actual and predicted values. In general, a lower RMSE is preferred because it means that the actual and predicted values are closer.


Mean Squared Error

MSE is another measure of the differences between the actual and predicted values. Once again, a lower MSE is preferred because it means that the predicted values are closer to the actual values.


R Squared

R2 is a decimal value that predicts what proportion of the variation in the target can be explained by the features. A higher R2 value is preferred because it means that a greater proportion of values can be predicted.

These graphs make it clear that the most accurate model for our problem is the linear ridge regression model, as this model has the lower RMSE (2.3) and MAE (0.87) as well as the highest R2 value (0.6623).

Conclusion

Our findings

Our results have confirmed that as all our inputs increase, excluding Damage Size and Damage Ratio, the size of the tumor will increase. In addition, we also found that the most accurate machine learning model for our data was the linear ridge regression model, as it had the highest R2 value. We hope that our findings and our website will help the healthcare industry in the identification and classification of melanoma tumors.

Scatter Matrix

A scatter matrix shows the correlation between all of our variables. It is very useful for getting a bird's-eye-view of our data and seeing which variables have strong correlations. One thing that sticks out with this graph is that none of our values have a particularly strong correlation with tumor size, which makes our task a little more difficult. This graph portrays each factor contributing to the size of each melanoma tumor, or each input, and how they increase or decrease exponentially according to the information from our dataset.

Scatter Plot

Scatter plots are useful in showing the correlation between two quantitative variables, in this case tumor size and malignant ratio. From this scatter plot, we can see that the correlation between these variables is positive and not very strong/moderately correlated.

Density Heatmap

This graph describes the amount of tumors that share a certain size and malignant ratio. The brighter squares signify a high number of tumors and the darker squares signify a lower number of tumors.

3D Model

The first graph depicts four values (malignant ratio, exposed area, area mass, and size appearance), and how they relate to one another. The orange and yellow dots signify an increase in size appearance and the purple and blue dots signify a decrease in size appearance. Similarly, the second graph depicts the relationships between four more values (malignant ratio, exposed area, damage size, and tumor size). The orange and yellow dots signify an increase in tumor size and the purple and blue dots signify a decrease in tumor size.

Violin Chart

This plot displays one input, damage size, and how often certain values of that input appear. The wider sections of the graph indicate a higher amount of a certain value, and the narrower sections of the graph indicate a lower amount of a certain value.

Parallel Coordinate Chart

This chart displays how each input changes as the tumor size increases or decreases. The green lines indicate an increase in tumor size and the red lines indicate a decrease in tumor size.

Correlation Heatmap

This heatmap is similar to the scatter matrix in the way that it displays relationships between the inputs and outputs. It is unique because of the way in which it displays direct correlations between each value. The lighter colors represent a greater correlation and the darker colors represent a lesser correlation.

Density Contour

On this graph we can see the distibution between the damage size variable and the tumor size, with the colored contours becoming brighter to portray the higher density of tumors at that contour. These two variables seem to be moderately correlated, showing that within a damage size between 75 and 55 there is a very dense group of tumors that lie in a size range of 6.5 and 10.5.
